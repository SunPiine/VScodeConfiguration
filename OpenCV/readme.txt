1.按mhtml 文件的内容进行配置
2.遇到编译失败，可以升级TDM_GCC 和Cmake 以及openCV 的版本，具体访问各自官网
3.在配置时，注意配置文件、代码中的版本数字，如435更改为openCV 所对应的版本号
4.如果使用本文件夹内程序进行安装的话，不要重命名本文件夹内的程序名，文件名包含了软件的版本号，这对于配置细节十分重要
5.进行配置之前请仔细参考mhtml 但不要完全照搬path 设置，并及时更新各组成软件
6.mthml 教程中使用的是MinGW 而本文件夹中是TDM_GCC 两者没有区别，均是GNU 的不同发布软件
7.tasks.json 是规定编译命令的文件，可以使用Crtl+Shift+P tasks 开始任务配置，添加头文件和库文件包含,需要认真配置，GCC族编译器的cpp.exe 工具不生成可执行文件，应当使用除此之外的其它编译工具，如c++.exe g++.exe 或其它在GCC bin 文件夹内的C++编译工具，出现文件未包含情况，如果确认的确已经设置好，可以直接编译(使用 shift + Ctrl + B )，因为有时VScode 有问题
8.c_cpp_properties.json 是cpp 编辑器配置；tasks.json 是编译器配置；launch.json 是调试器配置是默认的，或者是和一般的cpp调试配置一致
9.本文件夹内包含一个完整的opencv 项目实例和编译结果，可以参考其中的配置进行设置，切不可直接复制，配置和系统、编译软件的细节相关
